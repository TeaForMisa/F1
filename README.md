# 🏎 F1 Telegram Бот

Бот для Формулы-1 на **Python + Aiogram 3**. Присылает уведомления перед гонками,
квалификациями, спринтами и практиками — за 2 часа, 1 час и 30 минут до старта.
Все времена — в часовом поясе пользователя. Данные — из открытого Jolpica-F1 API.

## Возможности

- **🔔 Уведомления** за 2ч / 1ч / 30мин до каждой сессии (гонка, квал, спринт, практики)
- **🌍 Часовой пояс** — пользователь выбирает из списка или вводит свой (IANA)
- **⚙️ Настройка** — можно отключить любой из трёх интервалов
- **📊 Админ-панель** — статистика, список пользователей, рассылка, обновление кеша
- **🔒 Безопасность** — защита от HTML-инъекций, троттлинг спама, валидация ввода
- **💾 Память** — настройки, лог уведомлений и состояния диалогов в SQLite,
  переживают рестарты и пересборки контейнера (на Bothost — папка `/app/data`)
- **📋 Кнопка «Меню»** — все команды доступны из нативного меню Telegram
  (у админов в меню дополнительно видны админ-команды)

## Команды пользователя

| Команда | Описание |
|---|---|
| `/start` | Регистрация + выбор часового пояса |
| `/help` | Список команд |
| `/next` | Ближайшая сессия (с обратным отсчётом) |
| `/schedule` | Расписание гоночного уик-энда |
| `/standings` | Очки пилотов |
| `/constructors` | Очки команд |
| `/timezone` | Сменить часовой пояс |
| `/settings` | Включить/выключить интервалы уведомлений |
| `/cancel` | Отменить текущее действие (ввод пояса, рассылку) |

## Админ-команды (только для ADMIN_IDS)

| Команда | Описание |
|---|---|
| `/admin` | Список админ-команд |
| `/stats` | Статистика: всего/активных/новых пользователей, отправлено уведомлений |
| `/users` | Последние 20 зарегистрированных пользователей |
| `/user <id>` | Подробная информация о пользователе |
| `/broadcast` | Рассылка сообщения всем активным пользователям |
| `/refresh` | Принудительное обновление кеша сессий и зачётов |

Узнать свой Telegram ID: напиши [@userinfobot](https://t.me/userinfobot).

## ⚠️ ВАЖНО: чистый Git-репозиторий

В репозитории должны быть **ТОЛЬКО файлы бота** (`.py`, `requirements.txt`, `.env.example`).
Никаких файлов от Next.js/Node.js (`eslint.config.mjs`, `package.json`, `tsconfig.json`, `next.config.ts`, папок `src/`, `node_modules/` и т.д.) — иначе Bothost попытается их обработать как Python и упадёт с `SyntaxError`.

Файл `.gitignore` в репозитории уже настроен так, чтобы случайно не включить лишнее.

## Деплой на Bothost.ru

1. **Создай НОВЫЙ пустой Git-репозиторий** (GitHub/GitLab). Не используй репозиторий со старым Next.js-проектом!
2. **Залей в него содержимое папки `f1-bot/`** (все `.py` файлы, `requirements.txt`, `.env.example`, `.gitignore`, `README.md`, папку `handlers/`) — в корень репозитория.
3. На `bothost.ru/create-bot.php`:
   - Платформа: **Telegram**
   - Библиотека: **aiogram**
   - Bot Token: от @BotFather
   - Git URL: ссылка на твой репозиторий
   - Ветка: `main`
4. В панели Bothost → переменные окружения:
   - `ADMIN_IDS` = твой Telegram ID (обязательно для админки)
   - `DB_PATH` = `/app/data/f1bot.db` (обязательно — папка `/app/data` персистентна)
5. Bothost сам соберёт и запустит бота. Проверь «Логи работы» — должно быть `Бот запущен и готов к работе ✅`.

## Локальный запуск

```bash
cd f1-bot
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# вставь BOT_TOKEN и ADMIN_IDS в .env
python main.py
```

Требуется Python 3.10+.

├── handlers/
